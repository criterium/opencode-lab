# LLM Determinism: 3 Experiments

**Date:** Jun 4, 2026
**Base model:** The same LLM in all experiments
**Context:** Research on analytical determinism in language models

---

This report presents **3 determinism experiments** conducted on the same model with different designs, jointly characterizing its stability in analytical tasks:

- **Experiment 1 — Single-prompt replicates:** The simplest design: the same prompt is run 10 times with no evolution or accumulation. Measures pure model variance under identical conditions.
- **Experiment 2 — Chained forks:** 3 branches of the same model analyze the same 12 documents, measuring divergence when context and evolution are present.
- **Experiment 3 — Cross-model evaluation:** The most complex design: 4 different models answer the same 8 questions, measuring how much model switching adds to the existing variance.

## Experiment 1: Single-prompt replicates (10×)

### Design

The same model received the **same prompt 10 times** in independent sessions. No chain, no accumulation, no evolution. Each session evaluated the 4 determinism reports (generated by the fork analysis in Experiment 2) with 0-10 scores on content and structure.

**Exact prompt (from `research/deepseek-v4-flash-determinism/dump/README.solo_informes.prompt.md`):**

```
Evaluate the 4 determinism reports (research/deepseek-v4-flash-determinism/README.*.md) with numeric scores 0-10 in two dimensions:

## Content (average of 3 subfactors)
- Technical depth: whether it defines concrete metrics and calculates a numeric ID, or stays at qualitative descriptions
- Practical utility: whether it gives actionable recommendations or just describes the problem
- Autonomy: whether it applies its theoretical framework to the experiment data or stays as a generic disconnected essay

## Structure (average of 3 subfactors)
- Organization: clear sections, logical hierarchy, separation between analysis and conclusions
- Navigability: table of contents, appendices, ease of finding specific information
- Completeness: covers relevant aspects without redundancy or gaps

## Output format
Output a table like this and write the result to research/deepseek-v4-flash-determinism/dump/README.solo_informes.v1.es.md:

| Model | Content (0-10) | Structure (0-10) | Average |
|---|---|---|---|
| DeepSeek V4 Flash Free | ? | ? | ? |
| MiMo V2.5 Free | ? | ? | ? |
| MiniMax M3 Free | ? | ? | ? |
| Nemotron 3 Super Free | ? | ? | ? |

## Notes
- Read the 4 README files from the research/deepseek-v4-flash-determinism/ directory
- No other context is needed
- Scores must be numeric, not qualitative
- Briefly justify each score if possible
```

### Results

| Session | MiMo | DeepSeek | MiniMax | Nemotron |
|---|---|---|---|---|
| v1 | 9.17 | 9.00 | 8.33 | 3.83 |
| v2 | 8.7 | 7.5 | 7.0 | 3.3 |
| v3 | 9.33 | 9.33 | 7.67 | 4.83 |
| v4 | 9.0 | 8.7 | 6.7 | 3.0 |

| v6 | 8.5 | 8.7 | 7.5 | 4.3 |
| v7 | 8.5 | 8.7 | 7.4 | 4.2 |
| v8 | 8.7 | 9.0 | 7.8 | 4.8 |
| v9 | 9.0 | 8.7 | 7.8 | 4.5 |
| v10 | 9.3 | 9.2 | 8.3 | 5.5 |
| **μ** | **8.89** | **8.74** | **7.55** | **4.19** |

**Ordinal stability:**

| Position | Model | Agreement across 10 sessions |
|---|---|---|
| 1st | MiMo | **60%** (6/10) |

### Findings

1. **Low numeric determinism.** Average σ of 0.51 points. Two replicates can differ by up to 2.33 pts.
2. **High ordinal determinism.** Tail ranking (3rd and 4th) is 100% stable. Head ranking (1st vs 2nd) is stable 60% of the time.
3. **MiMo and DeepSeek are indistinguishable.** Mean difference of only 0.15 pts. Any preference is noise.
4. **Nemotron is the only clear case:** always last, with the highest variance (σ=0.72).

## Experiment 2: Chained forks (3 branches → 12 docs)

### Original objective

Measure and compare the behavior of the `high` and `max` variants of the `reasoning_effort` parameter in DeepSeek V4's Pro and Flash models, using an identical trivial prompt (*"Any relevant findings?"*), to determine whether the difference is consistent with official documentation and what implications it has on cost and reasoning patterns.

A ~63k token session on this comparison was forked into 3 branches. Each branch used a different combination (Flash Max, Pro High, Pro Max) and generated a response to the same prompt. The 3 responses were blindly labeled **Op1**, **Op2**, **Op3**.

### Determinism question

This led to a meta-analysis: what happens when the **same model** (DeepSeek V4 Flash Free) repeatedly analyzes the same data (Op1, Op2, Op3 and their derivatives)? Does it reach the same conclusions or diverge? To measure this, the same model was asked to analyze the resulting 12 documents in **3 independent forks**. If the model were deterministic, all 3 analyses would be identical. If not, they would diverge.

### Document chain

Each of the 3 forks generated a chain of 4 documents using single-blind methodology, detailed in the original research (`research/deepseek-v4-reasoning-effort-high-vs-max/`). The 3 analyst models (DeepSeek V4 Flash Free, MiMo V2.5 Free, MiniMax M3 Free) received the 12 resulting documents and each applied its own metric framework.

### Key results

| Metric | DeepSeek | MiMo | MiniMax |
|---|---|---|---|
| Correct assignments (blind phase) | 33%, 0%, 33% | 22% average | AS=0 |
| Jaccard between forks | 0.40-0.50 | ~35% level 0 | 0.73 v / 0.69 r |
| ID calculated | 38.6% | 0.53→0.97 per level | No composite ID |
| Post-revelation improvement | 100% | 22%→100% | AS=0→1.0 |

**Findings:**

1. **Low determinism in blind phase.** The 3 branches barely agree (~20-33% agreement). Each used different criteria for the same task.
2. **Total convergence with complete information.** When the truth is revealed, all 3 branches converge to 100%.
3. **Bifrontism: data vs interpretation.** The model is deterministic on numbers and technical facts, but indeterministic when interpreting them.
4. **Error propagation.** A numeric error (cost gap $0.000140 vs $0.000077) propagated through 6 of 8 forks with no spontaneous correction.
5. **Lost discriminator.** The data that would resolve the ambiguity (generation speed) was available but no branch used it.
6. **Every metric framework is complementary.** DeepSeek's ID (38.6%) measures the blind phase. MiMo's per-level ID measures improvement with information. MiniMax's bifrontism explains why both are right.

**Recommendations from the 3 models:**
- DeepSeek: Fork 2-3 instances, don't trust a single run.
- MiMo: Verify data between levels, use explicit ground truth.
- MiniMax: Force quantitative discriminants, report variance between forks.

### Quantitative deviation of the 3 forks

The most direct metric is **agreement on model+variant assignment for the 3 options** (Op1, Op2, Op3). Each fork made 3 decisions = 9 total decisions:

| Decision | Truth | Fork 1 (v1) | Fork 2 (v2) | Fork 3 (v3) |
|---|---|---|---|---|
| Op1 = ? | Flash Max | Pro High ❌ | Pro High ❌ | Pro Max ❌ |
| Op2 = ? | Pro High | Flash Max ❌ | Pro Max ❌ | **Pro High ✅** |
| Op3 = ? | Pro Max | **Pro Max ✅** | Flash ❌ | Flash ❌ |

**Hits per fork:** Fork 1 = 1/3 (33%), Fork 2 = 0/3 (0%), Fork 3 = 1/3 (33%)
**Total hits:** 2 out of 9 = **22%**
**Pairwise agreement:** v1∩v2=1/3, v1∩v3=0/3, v2∩v3=1/3 → average **22%**

Additionally, each fork discovered unique findings the others missed:

| Finding | Fork 1 | Fork 2 | Fork 3 |
|---|---|---|---|
| parentID bug in fork | ✅ | ✅ | ✅ |
| Reasoning preserved | ✅ | ✅ | ✅ |
| $0.00017 cache gap | ✅ | ❌ | ❌ |
| High/Max invisible in forks | ❌ | ✅ | ❌ |
| getUsage() verification | ❌ | ❌ | ✅ |

**Findings shared by all 3:** 2 out of 6 = **33% overlap** (Jaccard ~0.40 between pairs).

In summary: **3 forks of the same model on the same input agree only on 22-33% of key decisions and 40% of findings.** The remaining 60-78% is divergence attributable to the model's indeterminism on open analytical tasks.

### Majority voting paradox

If the 3 forks disagree, one might expect majority voting to improve the result. Here the opposite occurs:

| Decision | Fork 1 | Fork 2 | Fork 3 | Majority | Correct? |
|---|---|---|---|---|---|
| Op1 | Pro High | Pro High | Pro Max | **Pro High** | ❌ (was Flash Max) |
| Op2 | Flash Max | Pro Max | Pro High | **None** (1 each) | — |
| Op3 | Pro Max | Flash | Flash | **Flash** | ❌ (was Pro Max) |

**Majority vote: 0/3 correct**, while individually Fork 1 and Fork 3 scored 1/3 each. **Majority combination makes results worse.** This happens because errors are correlated: forks don't fail in random directions but share biases (overestimating Pro, underestimating Flash). The majority inherits the shared bias instead of canceling it.

The implication is direct: **forking is not enough; external information (ground truth) is needed to correct the shared bias.** Consensus among forks without revealed truth is no more reliable than a single fork — and can be worse.

### Divergent recalibration is echo, not new indeterminism

When the truth was revealed (Op1=Flash Max, Op2=Pro High, Op3=Pro Max), all 3 branches recalibrated and achieved 100% accuracy. However, each READMEr (r1, r2, r3) added different findings and organized its correction differently. Is this indeterminism? **Not really.**

The 3 branches did not start from the same point when recalibrating. Each arrived at READMEr with its own `modelov*.md → modelov*r*.md → READMEv*.es.md`, carrying 3 levels of accumulated divergence:

| Branch | Assignments in modelov | In modelovr | In READMEv | Upon reaching READMEr |
|---|---|---|---|---|
| **r1** | Op1=Pro High, Op2=Flash Max, Op3=Pro Max | Same | Same | **2 labels swapped** |
| **r2** | Op1=Pro High, Op2=Pro Max, Op3=Flash | Same | Same | **3 different labels** |
| **r3** | Op1=Pro Max, Op2=Pro High, Op3=Flash | Same | Same | **2 labels swapped** |

If 3 people receive the same correction but each wrote a different draft, it's expected that each corrects differently. **Divergent recalibration is not new indeterminism; it's the echo of what already existed in the first analytical decision.**

The critical moment of low determinism is **the first question** (`modelov*.md`), where all 3 branches started from the same prompt and context. There, divergence was total (22-33% agreement). Once each fork took a different path, everything else — recommendations, README, recalibration — is a consequence of that drift. Determinism does not worsen along the chain; it was already broken from the first link.

## Experiment 3: Cross-model evaluation (4 models, 8 questions)

### Design

Four *different* models (DeepSeek V4 Flash Free, MiMo V2.5 Free, MiniMax M3 Free, Nemotron 3 Super Free) answered the same **8 sequential questions** about the 12 documents from Experiment 2. Unlike Experiment 1, this does not measure determinism (which requires the same model), but rather **how much evaluation diverges when switching models**. If forks of the same model agree ~22-33%, different models diverge even more — and that difference quantifies how much model switching adds to the existing variance.

### Results

| Model | Average score (A1) | Coherence (A2) | Final report (A3) | σ intra-session |
|---|---|---|---|---|
| **DeepSeek** | 8.88 | 9.0 | 9.5 | 0.35 |
| **MiMo** | 7.75 | 8.0 | 9.5 | 0.89 |
| **MiniMax** | 7.81 | 6.5 | 9.0 | 0.38 |
| **Nemotron** | 5.44 | 2.5 | 4.8 | 2.39 |

**Intra-session stability:**

| Model | Score range (P1-P8) | Trend | σ |
|---|---|---|---|
| DeepSeek | 8.5-9.5 | **Stable →** | 0.35 |
| MiMo | 6.0-8.5 | **Variable ↑** (recovers after P3 error) | 0.89 |
| MiniMax | 7.5-8.5 | **Stable ↑** (linear improvement) | 0.38 |
| Nemotron | 0.0-7.0 | **Erratic ↓** (collapses in P5-P6) | 2.39 |

**Findings:**

1. **Cross-model divergence far exceeds within-fork divergence.** While in Experiment 2, three forks of the same model diverged ~22-33%, here 4 different models show qualitative differences on every question. Intra-session σ ranges from 0.35 (DeepSeek) to 2.39 (Nemotron) — much higher than between-replicate σ (~0.52 in Experiment 1).

2. **DeepSeek is the most coherent.** No ups and downs. Flat, high performance throughout.

3. **MiMo makes early errors but recovers.** Its worst moment is P3 (format error), but it climbs back to tie DeepSeek by P7-P8.

4. **MiniMax improves over time.** Starts behind MiMo, ends ahead. Nearly linear progression.

5. **Nemotron collapses.** Loses the thread at P5-P6 (English response + empty answer) and never fully recovers.

6. **Cross-model divergence is not predictable from intra-model divergence.** Knowing the model has low determinism (~38.6% ID) does not predict how different it will be from another model. Switching models introduces additional variance that does not scale linearly with the ID.

> Full details of this evaluation (methodology, weights, per-question scores) are in the main report: `research/opencod-zen-free-models/README.md`.

## Cross-experiment comparison

| Dimension | Exp 1: Replicates | Exp 2: Forks | Exp 3: Cross-model |
|---|---|---|---|
| Models evaluated | 4 (reports) | 3 (metrics) | 4 (behavior) |
| Replicates | 10 sessions | 3 forks | 8 questions |
| Measured variable | Between-session variance | Between-branch agreement | Intra-session coherence |
| Observed σ | 0.3-0.7 (points) | ~40-50% (Jaccard) | 0.4-2.4 (scores) |
| Ordinal determinism | **100%** on tail | Low (22-33% agreement) | High (DeepSeek=stable) |
| Dominant factor | Subject ordering | Information availability | Format attention |

**Cross-cutting conclusion:** Across all 3 experiments, the model shows the same pattern: **convergence on the coarse** (who is better, which data is correct, what is the rank order) and **divergence on the fine** (exact scores, assignment criteria, wording). Model determinism is not binary; it depends on task granularity.

## Global determinism index

Combining all 3 experiments into a single index provides a unified view of model determinism. Each experiment is normalized to a 0-1 scale (1 = maximum determinism) and averaged:

| Experiment | Base metric | Normalization | Sub-index |
|---|---|---|---|
| **1 — Replicates** | Average σ = 0.51 out of 10 | 1 − (σ / 3) | **0.83** |
| **2 — Forks** | Between-fork agreement = 28% | Direct agreement | **0.28** |
| **3 — Cross-model** | Average intra-session σ = 1.00 | 1 − (σ / 3) | **0.67** |

```
Global_ID = (0.83 + 0.28 + 0.67) / 3 = 0.59
```

| Range | Classification |
|---|---|
| 0.80-1.00 | Deterministic |
| 0.60-0.79 | Moderately deterministic |
| 0.40-0.59 | **Poorly deterministic** ← ID = 0.59 |
| 0.00-0.39 | Indeterministic |

The global ID of **0.59** classifies the model as **poorly deterministic** across the 3 experiments. This value is consistent with the model self-assessments: DeepSeek (38.6%) measured the worst case (blind forks), MiMo (68%) measured the best case (with complete information), and the global ID (59%) falls between them as a weighted average of the 3 designs.

The main lever for raising this ID is not switching models, but **adding information**: when the model knows the truth (Experiment 2's r phase), between-fork agreement jumps to 100%. Model determinism is more constrained by task ambiguity than by its internal stochasticity.

### How to mitigate low determinism: the user's role

Low determinism is not a verdict; it's an input for designing a better workflow. Responsibility lies with the user, not the model. The following strategies, combined, yield reliable results despite model variance:

**1. Merge reports from multiple models as the primary strategy.** This research session is its own demonstration: it did not start from a single iteratively corrected analysis, but launched 4 different models on the same problem, evaluated their reports, and consolidated them into a final result integrating everyone's findings. If you lack the time or criteria to evaluate a model's responses in detail and correct them one by one, the most efficient approach is to iterate transversally: launch several models or forks, generate a structured report or plan for each, then compare and consolidate. The result will always be richer, with less time and effort, than trying to perfect a single response through successive corrections.

**2. Fork across different models, not just within the same one.** Using a single model (even with 3 forks) inherits its shared biases. Combining different models — e.g., DeepSeek for depth and MiMo for speed — leverages their uncorrelated errors, increasing the probability of covering more angles. The report `research/opencod-zen-free-models/README.md` provides a detailed ranking of each free model's strengths and weaknesses to guide this decision.

**3. Design prompts to reduce ambiguity.** The more specific the prompt, the lower the variance. Including format constraints, explicit criteria, and output examples reduces the model's search space.

**4. Validate against ground truth when available.** Experiment 2's r phase showed that with complete information, the model converges to 100%. Including a verification step with the correct answer dramatically improves consistency.

**5. Use the final report as a quality filter.** Final report quality predicts overall model quality. Asking the model for an extensive report and evaluating its depth, accuracy, and autonomy allows selecting the best run without external ground truth.

**6. Accept divergence as information, not noise.** If 3 forks or 3 models disagree, that divergence is valuable: it indicates the problem has inherent ambiguity. An expert user exploits that diversity for a richer view, rather than seeking a single "correct" answer.

Ultimately, the LLM is a generation tool, not a truth tool. The most robust strategy is not to find the most deterministic model, but to **orchestrate multiple models and consolidate their outputs**. Judgment, selection, and synthesis come from the user.

## General limitations

- **Single base model.** Results may not generalize to other LLMs.
- **Modest sample sizes.** 3 forks, 8 questions, 10 replicates — insufficient for robust statistics.
- **Uncontrolled parameters.** Temperature and seed were not explicitly fixed.
- **Analytical tasks only.** Code generation, translation, and other modalities were not tested.
